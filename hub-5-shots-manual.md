# Hub 客户端 5 段视频手工操作手册

> 用 **MiniMax Hub** 网页端图生视频 (first_frame) 模式, 跑完 5 段关键镜头。
> 适合 3000 积分账号、无 API 配额 / 不愿调 API 的场景。

---

## 0. 入口

打开 **https://hub.minimaxi.com/** → 登录 → 顶部菜单 **Media / 视频生成 / Video Generation**

**模型选 `Hailuo-2.3`** (或 Seedance 2.0 Mini 35 折, 但 35 折模型走独立 quota, 不一定扣这 3000 积分 — 看页面提示)
**模式选 `图生视频` → `first_frame`** (用上传图作为视频首帧)

---

## 1. 操作流程 (每段通用)

```
1. 选择模型: Hailuo-2.3
2. 模式: 图生视频 → first_frame
3. 上传图片 (路径见下表)
4. prompt 框粘贴 (见下表)
5. duration: 6s
6. resolution: 1080P
7. 点生成, 等 1-2 分钟
8. 下载 mp4, 改名为 shot-XX.mp4 (见下表)
9. 放进 D:\YouTubeVideo\luhuo-mp4\public\videos\
10. 重复下一段
```

## 3. 完成后

把 **5 个 mp4 文件放到 `D:\YouTubeVideo\luhuo-mp4\public\videos\`** 后, 告诉我一声, 我会跑:

```bash
node scripts/build-video.mjs --skip-audio --skip-verify
```

直接出最终成片 `D:\YouTubeVideo\luhuo-mp4\out\luhuo-main.mp4` (~5-10 MB, 60s 1080P)。

---

## 4. 注意事项

- **文件名严格匹配**: `shot-01.mp4` `shot-04.mp4` `shot-06.mp4` `shot-10.mp4` `shot-12.mp4` (大小写不敏感)
- **mp4 格式**: Hub 下载下来就是 mp4, 不用转码
- **时长严格 6s**: 不要选 10s, 否则后续音频会错位
- **画面无文字**: prompt 里反复强调"不要出现文字", Hub 模型可能偶尔违规, 下载后肉眼看一眼, 有字幕/水印重跑
- **积分预算**: 1080P 6s ≈ 30-50 积分/段 × 5 段 ≈ 150-250 积分, 3000 积分足够, 不用担心超

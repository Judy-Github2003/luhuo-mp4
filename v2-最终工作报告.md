## 3. MiniMax API 折腾全过程（重点）

### 3.1 计费体系盘点
测试了 4 套独立计费：
1. **按量付费 group**：`sk-api-...` key → 余额 0（base_resp 1008）
2. **Token Plan Plus 档**（¥49/月）：`MINIMAX_API_KEY` 环境变量 → 周额度用完，5h 滚动窗口 + 周累计双限
3. **Token Plan API 加购**：新生成 `sk-cp-...` key → 仍走 Token Plan quota，HTTP 200 但 `code=2056 quota exceeded`
4. **Media Plan Starter**（¥69/月）：Hub 客户端订阅，独立积分池 10,000 积分/月，**API key 不共享**

### 3.2 关键技术决策
- **放弃 API 调用视频**：3 把 key 都 2056 或 1008，无法用脚本调
- **改走 Hub 客户端手工跑**：Media Plan 10000 积分到账后，前 3 段成功 + 后 2 段成功（中间 3000 积分用尽后用户续费 → Media Plan 10000 积分到账 → 跑完 shot-06/10）
- **种子轮：Remotion 静态渲染**，verify-assets.mjs 在 build 时写 `effectiveType`，缺失回退 color-block

### 3.3 占位段处理
剩余 7 段（shot-02/03/05/07/08/09/11）仍在用 v1 阶段生成的 14 KB 单色占位 mp4——这些是之前 ffmpeg 生成的色块。直接跑渲染时它们会显示成 5 秒单色块 + 中文字幕，看起来突兀。

## 5. 7 段占位补齐方案（请求 ChatGPT 建议）

### 方案 A：静态图替代视频（推荐 ⭐）
**思路**：
- ChatGPT 直接生成 7 张 16:9 高质量静态图（不再做视频）
- `shots.json` 改 `visual.type: "image"`，src 指向 `images/shot-XX.png`
- 重新跑 `node scripts/build-video.mjs --skip-audio --skip-verify`
- Remotion `<Img>` 静态满铺，**比色块优雅很多**

**优点**：
- 零视频积分消耗（静态图用 ChatGPT 出或 MiniMax Hub 图片生成）
- 渲染速度几乎不变（Remotion 静态图比视频轻）
- 7 张图能保证风格统一（都是 ChatGPT 同一 prompt 体系）

**缺点**：
- 静态图无运动，比视频略死板
- 但配合镜头缓慢推进（Remotion 内建 zoom in 动画），效果不会差

### 方案 B：继续 Hub 跑 7 段视频
**思路**：
- 7 段沿用 shot-06/10 模式：ChatGPT 出 7 张新首帧图 + 复用之前的 prompt 格式
- Hub 客户端跑 7 次（每次 1-3 分钟）
- 预计消耗 ~2500 积分（剩 7500 积分够用）

**优点**：
- 动起来，比静态图生动

**缺点**：
- 需要再让 ChatGPT 出 7 张图（耗时）
- 7 段 prompt 都要写（之前 5 段 prompt 是文化馆精心设计的）
- 影片整体 60s 全动可能视觉疲劳，反而没节奏感

### 方案 C：当前版直接验收
- 5 真 + 7 占位成片已能完整叙事
- 中文字幕、音频、整体调性都到位
- 占位段相对突兀，但作为"demo 版本"已经合格

**我个人倾向方案 A**——静态图替代视频，理由：
1. 这是一部**文化纪录短片**，节奏要稳，**不是动作片**，静态图反而有"博物馆展陈"的克制感
2. Remotion 可以加极轻的 zoom/pan 动画，让静态图有"呼吸感"
3. 节省积分，剩下 7500 积分留给后续真正需要的视频项目

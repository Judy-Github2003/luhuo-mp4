## 三、每条反馈的根因诊断（基于实测数据）

### 反馈 1：时间不对齐 + 黑屏

**实测数据**（5 段真视频 ffprobe）：

| 视频文件 | 实际时长 | shots.json 中对应时长 |
|---|---|---|
| `shot-01.mp4` | **5.875s** | start=0.04, end=3.33 → Sequence = 3.29s |
| `shot-04.mp4` | **5.875s** | duration=5s（推测 Sequence 5s） |
| `shot-06.mp4` | 未测（推测 5-6s） | duration=5s |
| `shot-10.mp4` | 未测（推测 5-6s） | duration=5s |
| `shot-12.mp4` | 未测（推测 5-6s） | duration=6s |

**根因分析**：
- **Sequence durationInFrames 不等于视频实际时长**——这是硬性时间错位
- 视频比 Sequence **长**（如 shot-01：5.875s vs 3.29s）→ 视频被 Sequence 截断（OffthreadVideo 在 Sequence 末尾冻结）
- 视频比 Sequence **短** → 视频播完后 last frame 定格（不黑屏但视觉停顿）
- **真正的黑屏可能来自**：
  - Sequence 末尾 fade-out 到 0 → 黑
  - 静态图 `<Img>` 在 Remotion 4 + Chrome headless 下偶发加载延迟
  - OffthreadVideo 解码失败
- **核心问题**：**shots.json 的 `start/end` 是 TTS 反推的音频时间**，但**视频时长应与音频时长一致**——目前 12 段 shots.json 中 `duration` 字段和实际视频时长**未对齐**

### 反馈 2：衔接垃圾，需要黑一下衔接

**根因分析**：
- 当前实现：`FadeInOut` 组件**只在单段内**做 opacity 0→1→0
- Sequence A 和 Sequence B **不重叠**，所以**没有"两段同时淡"的中间帧**
- 实际效果：A 末段淡出 → 黑 → B 首段淡入（中间是硬切到黑）
- 用户感受到"衔接垃圾"是因为：**没有真正的 fade-through-black 转场**
- **用户要的"黑一下衔接"** = **电影感的 fade to black + fade in**：
  - A 末段 fade-out 到黑
  - 短暂黑屏（~0.3-0.5s）
  - B 首段 fade-in

**修复方案**：需要 Sequence 重叠 + opacity 双向控制，或用 Remotion 原生 `<TransitionSeries>`

### 反馈 3：字幕未处于底部（v3 改动失效）

**根因分析（代码 bug）**：
- `BilingualSubtitle.tsx` 用了 `<AbsoluteFill>` + `style={{bottom: 24, ...}}`
- `<AbsoluteFill>` 默认 `position: absolute; top:0; left:0; right:0; bottom:0;` → 撑满整个画面
- 设置 `bottom: 24` 后，**box 高度 = 100% - 24 = 几乎全屏**
- 内部 `<div>` 用 `flex justifyContent: center; alignItems: center` 居中
- → 字幕显示在 **box 中央 = 画面中央偏下**
- **修改 `bottom=24` 没有让字幕贴底，反而让字幕框高度几乎不变**

**修复方案**：
- 把 `<AbsoluteFill>` 换成普通 `<div>`（position: absolute + 显式 bottom + 高度 auto）
- 外层 div 用 `position: absolute; bottom: 24; left: 0; right: 0; display: flex; justifyContent: center;`
- 这样外层 div 高度自适应，字幕会真正贴底

### 反馈 4：架构错误（12 个独立视频 + Remotion 拼接）

**当前架构**：
```
<LuhuoMain>
  <Sequence from=0 durationInFrames=99>  // shot-01
    <FadeInOut><ShotVideo visual={...} /></FadeInOut>
  </Sequence>
  <Sequence from=99 durationInFrames=150>  // shot-02
    <FadeInOut><ShotVideo visual={...} /></FadeInOut>
  </Sequence>
  ...
</LuhuoMain>
```
- 12 段 Sequence 串行铺满 60s，渲染出**单一 60s mp4**

**用户期望架构**：
```
shot-01.mp4 (独立 5s 视频)
shot-02.mp4 (独立 5s 视频)
...
shot-12.mp4 (独立 5s 视频)
↓
Remotion 拼接（Sequence 或 ffmpeg concat）→ 最终成片
```

**根因分析**：
- 当前架构每个 shot 共享一个 Remotion 渲染上下文，**视频时长由 Sequence 控制**
- 改成 12 独立视频后：**每个 shot 是独立 mp4，时长由视频本身决定，不再被截断**
- 拼接阶段才做时长对齐 + 转场
- **架构大改**：需要新增 per-shot 渲染脚本 + 拼接脚本
- **优点**：
  - 每个 shot 可独立审查、独立替换素材
  - 视频时长由内容决定，不被 Sequence 强行截断
  - 拼接灵活（可加转场 / 黑场）
- **缺点**：
  - 渲染复杂度上升（12 次 Remotion 调用）
  - 总渲染时间可能变长
